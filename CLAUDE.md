# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project

Analysis of token embeddings produced by a weather encoder trained on the ERA5 dataset. Current goal: implement a **k-means-based subspace clustering** algorithm to understand the structure of the tokens. Requirements:

- Must finish in **≤ ~6 hours** wall time on this machine.
- Must output the **subspace bases** (per-cluster basis vectors), not just cluster assignments.
- Token selection: planned approach is a parameter `k` that samples tokens uniformly at random across **all** latent files (not per-file).
- Open design decisions: how many SVD/power iterations to use for the per-cluster subspace estimation, or whether full PCA per cluster is preferable (e.g. `torch.pca_lowrank` / `torch.svd_lowrank` on GPU).

## Data

`latents_2/` — 1.2 TB, **13,021 files** `latent_{i}.pt` for `i = 0 … 13020` (complete, no gaps). Ignore `latents_2/scp_transfer.log`.

Each file is a pickled dict (load with `torch.load(path, map_location="cpu", weights_only=False)`):

| key | content |
|---|---|
| `idx` | scalar int64 tensor (sample index) |
| `latent` | float32 tensor of shape **`[1, 12288, 2048]`** |

Important details:

- The latent has a **leading batch dim of 1** — squeeze it; the real shape is `[12288, 2048]`.
- `12288` = HEALPix cells (nside=32 grid, 12·32²), one token per cell. `2048` = embedding dim per token.
- The cell index uses **NESTED HEALPix ordering** (confirmed 2026-06-12: the dominant-cluster world map is geographically coherent under NESTED, noise under RING — see `subspace_kmeans_runs/v6_subspace_big_d64/maps/map_annual.png`). Pure-numpy `healpix_nest2ring` / `healpix_ring_lonlat` helpers for cell→lon/lat live in `worldmap.py` (shared by the report + the temporal report).
- **Time axis:** the files are ERA5 states at **6-hourly cadence (4/day: 00/06/12/18 UTC)**. The stored `idx` is the integer sample index, *not* a timestamp — reconstruct the calendar date positionally as `datetime = 2014-01-01 00:00 UTC + idx×6h` (file 0 = 2014-01-01 00:00; file 13020 = 2022-11-30 00:00 — a full 9-yr 2014→2022 ERA5 range would reach 2022-12-31 18:00, ~127 more steps, so the last ~month of 2022 is not fully covered). `temporal_spatial.py` groups files into calendar months this way.
- One file ≈ 100 MB float32. The full dataset (1.2 TB) does **not** fit in RAM (512 GB), so any full-data pass must stream files. Total token count is 13,021 × 12,288 ≈ 160 M tokens.
- Sampling tokens "randomly across all latents" still requires opening every touched file (~100 MB read per file), so I/O dominates — group sampled token indices by file and read each file at most once.

## Hardware / Environment

- 48-core CPU, 512 GB RAM, 2× NVIDIA A100 40 GB.
- Python: `/usr/bin/python3` (no venv), PyTorch 2.6.0 with CUDA, both GPUs visible.
- No build/lint/test tooling exists; these are standalone scripts run directly: `python3 script.py` (long runs via `nohup`).
- **GPU peer-to-peer is silently broken on this machine**: `tensor.to()` directly between `cuda:0` and `cuda:1` returns zeros/garbage without any error (`can_device_access_peer` still reports True). Always route inter-GPU transfers through CPU (`.cpu()` then `.to(other_gpu)`), and do D2H copies from the thread that launched the producing kernels.

## Code

See **README.md** for full documentation. Always keep README.md and this section up to date when changing code (user requirement: every script must be documented).

- `cluster_io.py` — shared sampling/IO layer all clustering scripts (subspace_kmeans, and any k-means/k-center) import: `load_tokens()` (file sampling + parallel load + writes `sample.json`), `sample_fingerprint()`/`load_file_list()`, and `save_model()`/`save_assignments()` which enforce the cross-algorithm `model.pt`/`assignments.pt` schema (required: `U [K,2048,d]` with `d=0` for plain point clusters, `means`, `eigvals`, `trace`, `counts`, `explained_var_ratio`, `config["method"]`, `history`, `sampled_files`, `sample_fingerprint`; optional: `radius [K]` for k-center's minimax objective, `final_obj_per_token` = the objective/token of the *saved* model under its final relabel sweep — the true minimised residual holdout_eval.py compares the held-out objective against; absent in older runs, fall back to `history[-1]["obj_per_token"]`). Also exports the shared `OVERFIT_GAP_THRESHOLD = 0.03` overfit cutoff used by both `holdout_eval.py` and the report's held-out verdict. K-means is the `d=0` case of K-subspaces (the residual formula collapses to plain squared distance to centroid). `load_tokens`'s `--max-ram-gb` guard counts only the fp16 `data` buffer; true peak RAM also holds the int32 file/cell-id arrays and transient float32 decode buffers (~`load_workers` × 100 MB), so leave headroom.
- `subspace_kmeans.py` — K-subspaces clustering (assignment by orthogonal residual to affine subspaces; per-cluster bases from exact PCA via batched `eigh` of streaming-accumulated 2048×2048 covariances; both GPUs used via one thread per GPU, accumulators merged on CPU). Defaults: 1500 random files × all 12288 tokens (~75 GB RAM), K=64 clusters, d=16, ~7 min wall time. Saves via `cluster_io.save_model`/`save_assignments`. A final relabel sweep under the saved bases writes `assignments.pt` and reconciles `counts` to those final labels (so `model['counts'] == bincount(assignments label)`); that sweep's objective is stored as `final_obj_per_token` (the `history` entries are the *previous* model's objective, one step behind the saved model). Use `--files-from <dir>/sample.json` to cluster a new config on the **identical token sample** for direct comparison (runs sharing a fingerprint are comparable; same seed+tokens-per-file reproduces exactly). `--dim 0` runs plain k-means. Tiny clusters (`< max(2d,64)`) are re-seeded by **splitting the largest healthy cluster along its top PC** (both halves inherit its subspace, offset ±1 std along PC1); a plain random-token re-seed can't recover at large `d` because a fresh `U=0` point-seed can't out-compete the rivals' `d`-dim subspaces — this stranded 1 cluster at `d=64` / 2 at `d=128` in earlier runs (v4/v5). `--max-ram-gb` guards the fp16 RAM budget (default 350; a 7000-file sample is ~352 GB, so pass e.g. `--max-ram-gb 420`).
- `holdout_eval.py` — generalization check. Freezes a run's trained means/bases and replays the exact assignment residual (`‖x−μ_j‖²−‖Uⱼᵀ(x−μ_j)‖²`) on tokens from latent files **disjoint from `sampled_files`**, reporting held-out vs in-sample residual fraction (writes `<dir>/holdout.json`; `analyze_clusters.py` then renders a "Held-out generalization" section). Reuses `cluster_io.load_tokens` by handing it an explicit held-out file list (its manifest goes to a throwaway `<dir>/holdout/` subdir, not the run's `sample.json`). Single GPU. Verified v6 (K=128, d=64): held-out 31.5% vs in-sample 31.5% residual — subspaces generalize, not overfitting.
- `worldmap.py` — shared HEALPix geometry + cluster coloring + world-map renderer, imported by both `analyze_clusters.py` and `temporal_spatial.py`. Pure-numpy `healpix_nest2ring` / `healpix_ring_lonlat` (cell→lon/lat), validated data-free: `nest2ring` is a bijection of `[0,12288)`; lon/lat clean (lat ∈ [−88.5°, 88.5°]); equal-area cell distribution matches analytics (50.0% at |lat|<30°; 13.7% at |lat|>60° vs 13.4% theoretical). `build_affinity_matrix` (subspace-affinity [K,K] for d>0, else centroid cosine) and `affinity_ordered_colors` (Fiedler-vector spectral seriation → `turbo`, so similar clusters share hues). `get_coastlines()` fetches+caches the Natural Earth 110m coastline GeoJSON (stdlib `json`, no cartopy/shapely needed). `render_world_map()` draws a Mollweide map with two readability features: **continent outlines** and a smooth **heatmap** (per-cell RGB interpolated via `scipy.griddata` onto a regular grid + Gaussian-smoothed, painted with `pcolormesh` — replaces the old speckled 12,288-pixel scatter; ~0.8 s/map). RGB (not the categorical cluster id) is interpolated, valid because `affinity_ordered_colors` already makes similar clusters share colors.
- `analyze_clusters.py` — algorithm-agnostic Markdown report generator; reads any `model.pt`/`assignments.pt` following the `cluster_io.py` schema (convergence, variance decomposition, per-cluster spatial/temporal columns, subspace affinity when `d>0`, optional held-out section from `holdout.json`, token-sample fingerprint + reproduce command). The **world map and the temporal/seasonal analysis have moved** to the dedicated `temporal_spatial.py` report; this report keeps the compact per-cluster spatial/temporal columns and a pointer to `temporal_report.md`. Imports `build_affinity_matrix` (for the affinity table) from `worldmap.py`. Backfills `sample.json` for pre-manifest runs. `python3 analyze_clusters.py --dir <out_dir> --out <report.md>`. **Every report metric is defined/interpreted in `METRICS.md`** — keep it in sync.
- `temporal_spatial.py` — dedicated **temporal & spatial report** (`temporal_report.md`): 12 monthly dominant-cluster maps + an annual map (continent outlines + heatmap via `worldmap.py`), a monthly-enrichment table flagging seasonal clusters, and a month-to-month dominant-cluster flip-rate curve. Builds the file→calendar-month mapping from `START + idx×6h` and reuses the run's *existing* `assignments.pt` (v6 covers all 12,288 cells in every calendar month, ~528–623 files/month) so it needs **no recomputation** — a ~10 s rendering/reporting pass. Colors computed once (`affinity_ordered_colors`) and shared across all maps. `python3 temporal_spatial.py --dir <run>`.
- Completed runs live under `subspace_kmeans_runs/`, each `vI_<name>` numbered in chronological order: `v1_subspace_out` (1500 files, K=64, d=16); `v2_subspace_big` (7000 files, K=128, d=32, fingerprint `82ca602ed7e7`); `v3_subspace_big_i100` (same sample as v2, 100 iters); `v4_subspace_big_d64` (same sample as v2, d=64); `v5_subspace_big_d128` (7000 files, K=128, d=128); `v6_subspace_big_d64` (same sample as v2, K=128, d=64, **split-largest re-seed guard** — supersedes v4: all 128 clusters healthy, min 227,886 tokens vs v4's stranded singleton; identical variance 8.4/60.1/31.5%, max d80 drops 65→40); `v7_seed1_d64` / `v8_seed2_d64` (same sample as v2, init **seed** 1 / 2 — the seed-robustness sweep; the v6 config reproduced to confirm d=64/K=128 is a stable basin, not a lucky init). Each has a `report.md` (and a `run.log` for the longer runs). Note: at large K the dominant-cluster map looks "scattered" only due to colormap aliasing — neighbor-agreement metrics confirm structure improves with K (see `temporal_report.md` / README). `v6` additionally has `temporal_report.md` + `maps/map_annual.png` + `maps/map_month_{01..12}.png`. **Seed-robustness verdict (the final check):** across v6/v7/v8 the objective (1887.8/1892.3/1888.2, 0.24% spread), variance split (~8.4/60.1/31.5%), EVR/d80, and seasonal cycle shape all reproduce to ~0.5%; all 128 clusters stay healthy (min ≥193k, zero singletons); and the three spatial partitions agree with NMI 0.72 pairwise (vs 0.13 chance). The d=64/K=128 result is seed-stable → **v6 is the final config**. Sweep driver `run_seed_sweep.sh` (detached `setsid nohup`); log `seed_sweep.log`.
